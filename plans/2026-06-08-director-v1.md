# Director v1 — Implementation Plan

> **Status:** APPROVED — ready to execute; start at Phase 0. (Design validated; all open decisions resolved 2026-06-08.)

## Specification

**Goal:** Build the *visibility-first coordination core* — a Go `director` CLI (the only sanctioned writer of log/fleet surfaces), its Claude Code hook integration, and the model-facing protocol skill — proving the **LOG → deterministic projection** loop end-to-end across concurrent sessions. Reference: `docs/specs/2026-06-03-director-coordination-design.md` (the spec; §-refs below point into it).

**Scope decisions (read first):**
- **In v1:** the §5 hook-first core — CLI write path, identity, event store, fleet/liveness, render/brief/status, hooks + installer, the protocol skill, **adoption Tier 0+1** (`director adopt`, §6), and the §13 quality suite.
- **Deferred:** **Tier-2 brownfield fan-out** (§6 steps 2–4 — parallel code-mapping, doc living/record/rot reconciliation, arc42 synthesis, back-dated ADRs) → **immediate fast-follow**; Phase-3 monitor/reaper, notifications, freshness sweep, multi-machine sync, SQLite escape hatch → later. v1 ships adoption Tier 0+1 (operational floor + open-loop import), which makes Director usable on existing repos without the heavy fan-out. (Spec §6/§8/§11 reconciled 2026-06-08.)
- **Go dependencies:** stdlib-first — `flag` + manual subcommand dispatch (no cobra), `encoding/json`, `os`, `testing`. One vetted build-time dep: `github.com/oklog/ulid/v2` (pure-Go, **zero transitive deps**) for ULID correctness. A single static binary, no *runtime* deps (§15.1). Build-time deps are governed by the §8 policy — distinct from the runtime-plugin prohibition; each earns its place.

**Success Criteria** (the §13 quality scenarios — the gate for "done"):

- [ ] **No data loss:** zero lost entries under N concurrent `emit` writers, and across resume-after-compaction (§13 t1, t3)
- [ ] **Render determinism:** same inputs → byte-identical `render` *and* `brief`; `--verify` passes (§13 t4)
- [ ] **Identity stability:** one workstream keeps one id across resume/compaction; repo-key matrix resolves collision-free (§13 t2, t3)
- [ ] **Fail-safe:** a deliberately-broken hook never blocks session start; failure surfaces in `health/` (§13 t5)
- [ ] **Low ceremony:** the only manual step is ~3 CHARTER lines per project (§13)
- [ ] **Coexistence:** `director install` is an idempotent `_managedBy`-tagged merge; `uninstall` removes only tagged entries, leaving GSD/other hooks intact (§5.4)
- [ ] **Adoption on existing repos:** `director adopt` brings an existing repo (worktrees/remote and all) into the fleet in ~5 min — 3-line CHARTER + assisted import of current open loops (Tier 0+1) (§6)

## Phase 0 — Scaffold & event schema

_Foundational; everything depends on this. One agent, sequential._

### Task 0.1: Module, CLI skeleton, ULID

**Context:** none (greenfield); spec §5.3 (CLI surface), §10 (IDs), §8 (build-time dep policy), §15.1.

**Steps:**
1. [x] `go mod init`; `go get github.com/oklog/ulid/v2` (pure-Go, zero transitive deps — the first build-time dep, admitted under the §8 policy; pinned in `go.sum`). Add `.gitignore` for runtime dirs (`fleet/`, `health/`, volatile `projects/*/`) — **but keep `projects/*/log.ndjson` and `CHARTER.md` committable** (§15.6: only the durable log is committed; volatile fleet/heartbeat gitignored).
2. [x] `cmd/director/main.go`: `flag`-based subcommand dispatch for the public verbs (`emit`, `resolve`, `render`, `brief`, `status`, `register`, `heartbeat`, `done`, `adopt`, `install`, `uninstall`) and a hidden `_hook` verb group (§15.6). Unknown verb → usage + non-zero exit; hooks path must still never crash a session (see Phase 5).
3. [x] `internal/id/id.go`: thin wrapper over `oklog/ulid/v2` (`New()`/`Parse()`) so the rest of the code never imports the lib directly (keeps it swappable). Note §10: each `emit` is a short-lived process minting exactly one id → same-ms ordering is best-effort and ambiguous cross-machine order is escalated, not silently chosen.

**Verify:** `go build ./... && go test ./internal/id/` (round-trip + sort-stability tests).

### Task 0.2: Event schema + NDJSON marshal

**Context:** `internal/ulid/`; spec §5.2, §17 (canonical kinds), §15.2 (NDJSON), §15.6 (schema_version).

**Steps:**
1. [x] `internal/event/event.go`: `Event` struct — `id` (ULID), `schema_version` (int, =1 from event #1), `type` (`decision|open-item|handoff|note` — §17), `workstream`, `area`, `risk` (`low|escalate`, decisions + needs-you open-items), `status` (`open|closed`, open-items), `addressed_to`, `refs` ([]ulid), `ts` (display-only), `body`. JSON tags = NDJSON line.
2. [x] Validation: reject unknown `type`; `risk` only on decision/open-item; `status` only on open-item; `refs` ULIDs well-formed.
3. [x] Table test covering each kind + the close-marker shape (`type: open-item, status: closed, refs:[id]` — §17).

**Verify:** `go test ./internal/event/ -run Schema`

## Phase 1 — Identity (repo-key + workstream-id)

_Foundational; the event store & fleet key on it. One agent, sequential. Independent of Phase 0.2._

### Task 1.1: Canonical repo-key with fallback chain + matrix test

**Context:** spec §4.3 (the algorithm + required test matrix), §15.6 (derive-once-then-read-persisted).

**Steps:**
1. [x] `internal/identity/repokey.go`: fallback chain — (1) `git rev-parse --git-common-dir` → collapse worktrees to one `.git`, slug abs path; (2) if remote exists → normalized remote URL (fork-vs-upstream collapse); (3) remoteless → common-dir path slug; (4) **persist** chosen key at `.director/repo-key`, and on subsequent runs **read the persisted key** (derive-once) so branch rename / remote change can't shift it.
2. [x] Shell out to `git` via `os/exec` behind a small seam (injectable for tests).
3. [x] **Matrix test (§13 t2):** linked worktree · no-remote repo · fork-vs-upstream · same-basename-different-repo → stable, collision-free keys.

**Verify:** `go test ./internal/identity/ -run RepoKey`

### Task 1.2: Workstream id + handle

**Context:** `repokey.go`; spec §4.2, §15.4/§15.6.

**Steps:**
1. [x] `internal/identity/workstream.go`: workstream-id = `repo-key + worktree-branch`, derived deterministically, persisted at `.director/workstream-id`; a resumed session re-derives the **same** id. Volatile session-uuid + heartbeat are *fields*, not the key.
2. [x] Human handle `<repo>-<branch>-<shortid>` for filenames/render/notifications.
3. [x] Test: same worktree+branch → same id across two derivations (simulates resume, §13 t3 half); branch rename keeps id when persisted.

**Verify:** `go test ./internal/identity/ -run Workstream`

## Phase 2 — Event store & write path

_Depends on Phase 0–1. Independent of Phase 3 (different store) → can run parallel with it._

### Task 2.1: NDJSON append store (the only writer)

**Context:** `internal/event/`, `internal/identity/`; spec §4.1 (write-path invariant), §15.2 (NDJSON/O_APPEND), §15.7 (atomic≠durable).

**Steps:**
1. [ ] `internal/event/store.go`: append one event as one JSON line to `projects/<repo-key>/log.ndjson` via `os.OpenFile(O_APPEND|O_CREATE|O_WRONLY)`, single `Write` of a `\n`-terminated line (line-sized atomic append on POSIX). Optional `fsync` flag for power-loss safety (§15.7).
2. [ ] Read side: `Tail(n)` and `ReadAll` iterators (used by render); stream-parse, don't slurp.
3. [ ] **Concurrency smoke test (§13 t1):** N goroutines/processes append simultaneously → assert **zero lost lines** and every line valid JSON. This is the test that retires the §4.1 fear for the Go writer.

**Verify:** `go test ./internal/event/ -run Append -race`

### Task 2.2: `emit` + `resolve` commands

**Context:** `internal/event/`, `internal/identity/`; spec §5.3, §17 (kinds + close-marker), §15.6 (resolve validation).

**Steps:**
1. [ ] `director emit --type <k> --area <a> [--risk …] [--to …] [--refs …] <body>` — derive workstream, mint ULID, validate, append. The **only** sanctioned write (forbid Edit/Write on the log surface — documented, enforced by convention + the skill).
2. [ ] `director resolve <ulid>` — append a close-marker (`type: open-item, status: closed, refs:[ulid]`). Validate the target ULID exists and is an open `open-item`; reject invented/closed/double-close (§15.6: resolve targets a CLI-surfaced ULID the model copies, never invents).
3. [ ] Wire both into `cmd/director` dispatch.

**Verify:** `go test ./internal/event/ -run Emit && go build ./... && bin/director emit --type note --area test "hello" && bin/director render --project <key>` (manual smoke)

**Steps:**

2. [ ] `director register` / `heartbeat` / `done` — row create / touch / terminal-archive (`fleet/archive/<date>/`, never delete, §5.5).
3. [ ] Drop self-set `idle` (§5.5): a row only carries heartbeat + terminal markers.

### Task 3.2: Derived liveness (TTL/lease)

**Context:** `internal/fleet/`; spec §15.4, §5.5.

**Steps:**
1. [ ] Liveness is **derived, never stored**: `active`/`stale`/`abandoned` from heartbeat-age (TTL) + whether the worktree/branch still exists / is merged (covers crash cleanup — a dead session stops heartbeating → stale).
2. [ ] Collapse multiple session-uuid rows per workstream at read time (the reaper itself is Phase-3, deferred — v1 only *derives & displays* liveness).
3. [ ] Test: aged heartbeat → stale; missing branch → abandoned; two uuids one workstream → collapsed.

## Phase 4 — Projections: render / brief / status

_Depends on Phase 2 (events) + Phase 3 (fleet). One agent, sequential._

### Task 4.1: The deterministic fold

**Context:** `internal/event/`, `internal/fleet/`; spec §5.3, §15.5, §10.

**Steps:**
1. [ ] `internal/render/fold.go`: sort by ULID; apply resolve/supersede markers (open-set for open-items, supersession resolution for decisions); read **tail + open-set** (not whole log); `--since <id>` spans active + archive. **Same inputs → identical in-memory projection.**
2. [ ] Ambiguous cross-machine ordering → flag for human, never silently pick (§10).
3. [ ] Determinism unit test: shuffle input order → identical fold output.

**Verify:** `go test ./internal/render/ -run Fold`

### Task 4.2: `render` (machine digest + manifest), `status` (cockpit), `brief` (human)

**Context:** `internal/render/`; spec §5.3, §9 (manifest), §16 + §17 (brief composition), §13 t4.

**Steps:**
1. [ ] `director render [--project <key>] [--verify]` — bounded digest used by SessionStart; emits a **manifest** (sources read, counts, last-verified ids) to `health/` (§9); `--verify` re-folds and asserts byte-identical (§13 t4).
2. [ ] `director status` — one-line-per-workstream cockpit (handle · status · recency · blocked-on, where blocked-on = open `open-item`s with `risk: escalate`); Needs-you band hard cap + "+N more" (§15.6).
3. [ ] `director brief [--project <key>]` (§16) — composes CHARTER outlook + latest handoff per workstream + open-item open-set (`risk:escalate`=needs-you) + decisions-since-last-review; fleet altitude when `--project` omitted. Deterministic (`--synthesize` deferred, §11).
4. [ ] **Byte-identical test for render AND brief (§13 t4).**

**Verify:** `go test ./internal/render/ && bin/director brief` (manual smoke)

## Phase 5 — Hooks & installer

_Depends on CLI commands (Phase 2–4). One agent, sequential._

### Task 5.1: CC hook adapter + handlers

**Context:** `internal/`; spec §5.4, §4.4 (layers), §14.2 (Ground Truth), §15.7 (isolate behind adapter, degrade gracefully).

**Steps:**
1. [ ] `internal/hook/adapter.go`: thin adapter over the CC hook JSON contract (stdin event → typed struct), isolated so a contract change is one-file; **every handler exits 0 on error** and logs loudly to `health/` (§5.4, §13 t5). Hidden `director _hook <event>` verbs.
2. [ ] **SessionStart** (incl. `matcher: compact`): derive workstream → `register`/`heartbeat` → inject CHARTER + bounded digest, **framed as authoritative current state — "build on it, don't rebuild it"** (Ground Truth, §14.2/§4.4); re-inject after autocompaction; filter subagent/throwaway sessions.
3. [ ] **PostToolUse:** fill-threshold "flush now while healthy" nudge (§4.4 layer 2).
4. [ ] **Stop/SessionEnd:** mark fleet status + run the **emit-guard** (block-with-correction when a turn looks like it made a decision/open-item/handoff but never emitted; respect `stop_hook_active` + wrap-up escape; nudges, never flushes — §4.4 layer 2).
5. [ ] `hooks/` shell shims that call `director _hook <event>` (so settings.json points at stable shims).

**Verify:** `go test ./internal/hook/ -run Handlers` + a fail-safe test feeding malformed input → exit 0, `health/` shows the failure (§13 t5).

### Task 5.2: Idempotent `_managedBy` installer

**Context:** `internal/install/`; spec §5.4 (the merge mechanism).

**Steps:**
1. [ ] `director install` — idempotent merge of Director's hook entries into `~/.claude/settings.json`, each tagged `_managedBy: "director"`; re-run is a no-op on present entries; never touches untagged / other-plugin (GSD) hooks.
2. [ ] `director uninstall` — remove **only** `_managedBy:"director"` entries.
3. [ ] Test against a fixture settings.json containing pre-existing GSD hooks → assert they survive install/uninstall round-trip.

**Verify:** `go test ./internal/install/`

## Phase 6 — The protocol skill (model-facing)

_Depends on the emit verbs existing (Phase 2). Can run parallel with Phase 4–5. One agent._

### Task 6.1: `skills/director/SKILL.md`

**Context:** spec §4.4 (layer 1 — the load-bearing flush habit), §17 (the 4 kinds), §14.2 (Ground Truth read-side), §15.6 (resolve via surfaced ULID).

**Steps:**
1. [ ] Write the protocol: **continuous boundary-flush** — emit `decision`/`open-item` the moment they arise; emit a `handoff` (current task · next action · hypotheses) at each natural boundary; a deferred loop is its own `open-item` (not packed into the handoff).
2. [ ] When-to-use each of the 4 kinds, with the `risk: escalate` rule for needs-a-human open-items.
3. [ ] **Ground Truth read-side instruction:** treat the SessionStart-injected CHARTER + digest as authoritative current state — build on it, do not re-derive/re-read.
4. [ ] `resolve` only against a ULID surfaced by the CLI (copy, never invent).
5. [ ] Frontmatter (name/description) so it activates on coordination/handoff context.

**Verify:** manual review against §4.4 + §14.2; (no code test — it's protocol text). A fresh session reading it can correctly emit + resolve in the Phase-7 e2e.

## Phase 7 — Adoption (Tier 0+1)

_Depends on Phase 1 (identity), Phase 2 (emit), Phase 3 (fleet). Makes Director usable on **existing** repos — the critical path for a brownfield reality. Can run parallel with Phase 4–6 once P1–P3 land. One agent. Spec §6 (tiered; v1-line note)._

### Task 7.1: `director adopt <repo>` — operational floor (Tier 0)

**Context:** `internal/identity/`, `internal/fleet/`; spec §6 (default adoption), §5.1.

**Steps:**
1. [ ] Derive repo-key + workstream-id for an **existing** repo (reuses the §4.3 chain — already handles worktrees / remotes / forks); create `projects/<repo-key>/` + a `CHARTER.md` from an interactive ~3-line stub (goal · non-goals · risk-line); `register` in the fleet.
2. [ ] Idempotent: re-adopting re-derives the same identity and **does not clobber** a human-edited CHARTER.
3. [ ] Test: adopt a fixture existing repo → identity stable, CHARTER scaffolded, fleet row present; re-adopt → no clobber.

**Verify:** `go test ./internal/adopt/ -run Floor`

### Task 7.2: Assisted open-loop import (Tier 1)

**Context:** `internal/adopt/`, `internal/event/`; spec §6.5 (seed above a relevance bar), §17 (open-item is the canonical home for deferred), §12 (relevance bar starts simple).

**Steps:**
1. [ ] Scan the repo for **open-loop candidates** — `TODO`/`FIXME`/"deferred"/checklist items in tracked files (and an existing `NOTES`/`MEMORY`/`*.md` if present). Bounded grep-style. **No code-mapping, no doc living/record/rot classification — that is Tier 2 (deferred, fast-follow).**
2. [ ] Present candidates; the human confirms which become `open-item`s; emit the confirmed ones via the Phase 2 write path so they land in the LOG's one home (consolidating the MEMORY-vs-docs scatter — §17).
3. [ ] Relevance bar is dead-simple in v1 (show all, human picks); §12's heuristic is a later refinement.
4. [ ] Test: seed a fixture with TODOs → candidates surfaced; confirmed → `open-item` events present; `brief` shows them.

**Verify:** `go test ./internal/adopt/ -run Import`
